# 开发准备工作

本文档详细记录了在开始开发家庭常用药管理系统代码之前需要完成的准备工作，确保项目开发的顺利进行。

## 1. 技术环境搭建

### 1.1 前端环境

1. 运行时: Node.js 20.x (LTS)。确保 Vite 构建工具的性能最优。
2. 构建工具: Vite 5.x。比 Webpack 更快，热更新（HMR）体验极佳。
3. 框架: Vue 3.4.x (Composition API)。利用其性能提升和更好的 TypeScript 支持。
4. UI 组件库: Element Plus 2.7.x。包含完善的表格、表单和日期选择器组件。
5. 状态管理: Pinia 2.1.x。Vuex 的替代者，逻辑更简单清晰。

- **安装Node.js**
  - 建议版本：20.x(LTS)
  - 下载地址：https://nodejs.org/en/download/
  - 验证安装：`node -v` 和 `npm -v`

- **创建前端项目**
  ```bash
  # 使用Vite初始化Vue3项目
  npm create vite@latest frontend -- --template vue-ts
  cd frontend
  ```

- **安装依赖**
  ```bash
  npm install element-plus pinia vue-router axios
  npm install --save-dev @types/node
  ```

- **配置开发环境**
  创建 `.env.development` 文件：
  ```env
  VITE_API_BASE_URL=http://localhost:8080/api
  ```

### 1.2 后端环境

1. **JDK**: **OpenJDK 17** (LTS)。Spring Boot 3.x 官方推荐版本，企业级标准，兼容性最好。
2. **框架**: **Spring Boot 3.5.14**。当前最稳定的 3 系列版本。
3. **持久层**: **MyBatis-Plus 3.5.x**。能够极大简化 CRUD 操作，非常适合 MVP 阶段的快速开发。
4. **数据库**: **MySQL 8.0**。利用其 JSON 字段支持，方便存储 OCR 的原始识别结果。
5. **安全鉴权**: **jjwt 0.12.x** + 手动拦截器实现。轻量级JWT方案，无需Spring Security。

- **安装JDK**
  
  - 建议版本：25 (LTS)
  - 下载地址：https://adoptium.net/
  - 验证安装：`java -version`
  
- **安装Maven**
  - 建议版本：3.8.x
  - 下载地址：https://maven.apache.org/download.cgi
  - 验证安装：`mvn -version`

- **创建Spring Boot项目**
  - 使用Spring Initializr：https://start.spring.io/
  - 选择依赖：Spring Web、MySQL Driver、MyBatis Plus、jjwt
  - 解压到 `backend` 目录

- **配置开发环境**
  修改 `application.yml` 文件：
  ```yaml
  spring:
    datasource:
      url: jdbc:mysql://localhost:3306/medmanager?useSSL=false&serverTimezone=UTC
      username: root
      password: your_password
      driver-class-name: com.mysql.cj.jdbc.Driver
  
  mybatis-plus:
    mapper-locations: classpath:mapper/*.xml
    type-aliases-package: com.example.medmanager.entity
  
  jwt:
    secret: your_jwt_secret_key
    expiration: 86400000
  ```

## 3. 版本控制设置

- **初始化Git仓库**
  ```bash
  git init
  ```

- **配置.gitignore文件**
  创建 `.gitignore` 文件：

- **设置分支**
  ```bash
  # 创建主分支
  git checkout -b main
  
  # 创建开发分支
  git checkout -b develop
  ```

- **配置CI/CD**
  创建 `.github/workflows/frontend-ci.yml` 文件：
  ```yaml
  name: Frontend CI
  
  on:
    push:
      branches: [ develop, main, 'feature/*' ]
    pull_request:
      branches: [ develop, main ]
  
  jobs:
    build:
      runs-on: ubuntu-latest
      
      steps:
        - uses: actions/checkout@v3
        - name: Set up Node.js
          uses: actions/setup-node@v3
          with:
            node-version: 16
        - name: Install dependencies
          run: cd frontend && npm install
        - name: Run ESLint
          run: cd frontend && npm run lint
        - name: Build
          run: cd frontend && npm run build
  ```

  创建 `.github/workflows/backend-ci.yml` 文件：
  ```yaml
  name: Backend CI
  
  on:
    push:
      branches: [ develop, main, 'feature/*' ]
    pull_request:
      branches: [ develop, main ]
  
  jobs:
    build:
      runs-on: ubuntu-latest
      
      steps:
        - uses: actions/checkout@v3
        - name: Set up JDK
          uses: actions/setup-java@v3
          with:
            java-version: '11'
            distribution: 'adopt'
        - name: Build with Maven
          run: cd backend && mvn clean package -DskipTests
  ```

## 6. 安全配置

- **生成JWT密钥**
  ```bash
  # 使用openssl生成随机密钥
  openssl rand -base64 32
  ```

- **配置密码加密**
  - 使用 Hutool 的 BCrypt 或自定义加密工具（无需Spring Security）
  - 推荐在 `common.utils` 包下创建 `PasswordEncoder` 工具类

- **设置CORS**
  创建 `CorsConfig.java`：
  ```java
  @Configuration
  public class CorsConfig implements WebMvcConfigurer {
      @Override
      public void addCorsMappings(CorsRegistry registry) {
          registry.addMapping("/**")
                  .allowedOriginPatterns("*")
                  .allowedMethods("GET", "POST", "PUT", "DELETE", "OPTIONS")
                  .allowedHeaders("*")
                  .allowCredentials(true);
      }
  }
  ```

## 9. 部署准备

- **配置部署脚本**
  - 前端部署脚本：`scripts/deploy-frontend.sh`
  - 后端部署脚本：`scripts/deploy-backend.sh`

- **设置环境变量**
  - 开发环境：`.env.development`
  - 测试环境：`.env.test`
  - 生产环境：`.env.production`

- **配置Nginx**
  创建 `deploy/nginx.conf` 文件：
  ```nginx
  server {
      listen 80;
      server_name med-manager.example.com;
      
      location / {
          root /path/to/frontend/dist;
          index index.html;
          try_files $uri $uri/ /index.html;
      }
      
      location /api {
          proxy_pass http://localhost:8080;
          proxy_set_header Host $host;
          proxy_set_header X-Real-IP $remote_addr;
          proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
      }
  }
  ```

- **准备Docker配置**
  创建 `Dockerfile` 文件：
  ```dockerfile
  FROM openjdk:11-jdk-slim
  COPY target/med-manager.jar app.jar
  EXPOSE 8080
  ENTRYPOINT ["java", "-jar", "app.jar"]
  ```

  创建 `docker-compose.yml` 文件：
  ```yaml
  version: '3'
  services:
    backend:
      build: .
      ports:
        - "8080:8080"
      depends_on:
        - db
    db:
      image: mysql:8.0
      environment:
        MYSQL_ROOT_PASSWORD: your_password
        MYSQL_DATABASE: medmanager
      volumes:
        - mysql-data:/var/lib/mysql
  volumes:
    mysql-data:
  ```

## 12. 常见问题与解决方案

- **数据库连接失败**
  - 检查数据库服务是否启动
  - 检查连接参数是否正确
  - 检查数据库用户权限

- **前端请求后端接口失败**
  - 检查后端服务是否启动
  - 检查CORS配置
  - 检查API基础地址配置

- **OCR识别失败**
  - 检查OCR服务API密钥
  - 检查图片格式是否支持
  - 检查网络连接

- **部署后服务无法访问**
  - 检查服务器防火墙设置
  - 检查Nginx配置
  - 检查服务是否正常启动
