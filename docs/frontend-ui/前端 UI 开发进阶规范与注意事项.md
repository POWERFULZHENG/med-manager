在前端 UI 开发过程中，除了基础的页面布局和视觉设计，**交互逻辑的严谨性**、**移动端的操作体验**以及**异常场景的处理**是决定系统“好不好用”的关键。

以下是为你补充的《前端 UI 开发进阶规范与注意事项》，建议将其集成到你的需求分析与实现方案 MD 文档中。

------

# 家庭常用药管理系统：前端 UI 开发进阶指南

## 1. 核心交互与体验 (UX) 增强

### 1.1 移动端优先的交互设计

由于药品管理通常发生在药箱旁，手机是第一操作终端。

- **触控友好**：按钮高度不低于 **44px**，列表项之间的点击区域需清晰，避免误触。
- **扫码/拍照优化**：OCR 上传应调用原生相机，并在界面提供“手电筒”提示（用于光线不足的药箱角落）。
- **抽屉式交互 (Drawer)**：在移动端，使用 `el-drawer` 代替 `el-dialog`，以便于单手操作。

### 1.2 搜索与过滤逻辑

- **即时搜索**：药品列表支持拼音首字母搜索（如输入 "AMXL" 匹配 "阿莫西林"）。
- **空状态设计**：当搜索无结果或库存为空时，展示友好的插画引导（如：“暂无此类药品，点击‘入库’添加”）。

------

## 2. 业务安全与防错机制

### 2.1 关键操作“二次确认”

- **出库校验**：出库数量严禁超过当前批次库存。若用户尝试出库全部数量，应提示“此操作将导致该批次库存清零”。
- **过期药拦截**：在出库界面，若用户选择了“已过期”的药品批次，UI 应弹出**强警告（红色背景）**，并提示：“该药品已过期，建议销毁，请确认是否继续领取？”。

### 2.2 表单鲁棒性

- **日期逻辑**：入库时的“失效日期”必须早于“生产日期”（若有），且必须晚于“当前日期”。
- **单位统一**：在输入数量时，动态显示单位（如：“请输入出库数量 (片)”），防止单位混淆。

------

## 3. 组件化开发规范 (Vue3 + Element Plus)

### 3.1 统一的状态标签 (Status Tag)

建议封装全局组件 `MedStatusTag`，统一管理状态颜色：

- `type="success"`: 正常（有效期 > 90天）
- `type="warning"`: 临期（有效期 < 30天）
- `type="danger"`: 过期（有效期 < 0天）

### 3.2 响应式表格 (Responsive Table)

- **PC 端**：展示完整列（批次、厂家、存放位置、效期）。
- **移动端**：使用 `v-if` 或 CSS 媒体查询隐藏次要列，只保留“名称”和“效期”，点击行展开查看详情。

------

## 4. 性能与加载优化

### 4.1 骨架屏 (Skeleton)

在药品列表加载过程中，使用 `el-skeleton` 展示占位图，减少用户焦虑感。

### 4.2 图片压缩

前端在上传 OCR 图片前，应利用 `canvas` 进行本地压缩，减少网络传输流量和后端识别延迟。

------

## 5. 前端安全注意事项

- **敏感数据遮蔽**：在用户管理界面，手机号等敏感信息应使用星号遮蔽（如 `138****8888`）。
- **JWT 自动续期**：在 `axios` 拦截器中处理 `401` 异常，若 Token 过期，自动引导至登录页。
- **防抖与节流**：入库点击按钮、搜索输入框必须增加防抖（Debounce），防止重复提交产生冗余批次。
